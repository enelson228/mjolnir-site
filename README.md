# MJOLNIR

A self-hosted, static personal site with a **cyber lab × Halo-inspired** aesthetic.

Built for reliability and low operational overhead: everything is plain HTML/CSS/JS, easy to deploy on a VPS, and easy to harden behind Nginx.

## What’s Included

- **Home** (`index.html`) — MJOLNIR landing page + hero mark.
- **About** (`about.html`) — “dossier”-style about page.
- **Projects** (`projects.html`) — project cards (renders from `projects.json` when present).
- **Telemetry** (`telemetry.html`) — client-side view that reads `/telemetry.json`.
- **NelsonWeb wrapper** (`nelsonweb.html`) — embeds a separate app under `/nelsonweb/` in an iframe.
- **Docs / Drop Zone** (`docs.html`) — links files placed under `/drops/` via `/drops/index.json`.

## Design Goals

- **Static-first**: ship as a static site, minimize attack surface.
- **Readable UI**: neon diagnostics, clear hierarchy, minimal clutter.
- **Deploy anywhere**: no build step required.
- **Hardening-friendly**: headers and caching rules are designed to work well with Nginx.

## Local Preview

From the repo root:

```bash
python3 -m http.server 8080
```

Open:
- http://localhost:8080/

## Production Deployment (Nginx)

Copy the site to a web root:

```bash
sudo mkdir -p /var/www/mjolnir
sudo rsync -av --delete ./ /var/www/mjolnir/
sudo chown -R www-data:www-data /var/www/mjolnir
```

### Example server block

An example Nginx config is provided in:
- `nginx-mjolnir.conf`

Notes:
- The live install for `nellysupreme.cloud` binds to a specific public IP to avoid a Tailscale `:443` conflict.
- `/telemetry.json` and `/projects.json` are expected to be generated by server-side jobs (not committed).

## Docs / Drops

- Server path: `/var/www/mjolnir/drops/`
- Public base URL: `/drops/`

This repo keeps the **index** (`/drops/index.json`) and a placeholder (`.gitkeep`), but ignores the actual drop contents in git.
